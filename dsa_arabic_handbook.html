الترتيب غير المقارن

Counting Sort

التعريف: لا يقارن العناصر — يعدّ تكرارات كل رقم ويستخدم التراكم لتحديد المواضع.

الخطوات:

  1. ابحث عن maxVal
  2. أنشئ مصفوفة count بحجم maxVal + 1
  3. عُدّ تكرار كل رقم
  4. حوّل إلى تراكمي (Cumulative)
  5. ضع العناصر في مصفوفة output بالترتيب العكسي
📌
نقطة امتحانية

Counting Sort هو Stable إذا مررنا على المصفوفة الأصلية من اليمين للشمال عند وضع العناصر.

Radix Sort

التعريف: يرتّب الأرقام رقماً رقماً من الأقل أهمية (LSD) للأكثر أهمية (MSD).

العلاقة: يستخدم Counting Sort كخوارزمية فرعية لكل خانة (آحاد → عشرات → مئات...).

مثال: [170, 45, 75, 90, 802]
آحاد: [170, 90, 802, 45, 75]
عشرات: [802, 45, 170, 75, 90]
مئات: [45, 75, 90, 170, 802] ✓

خوارزميات البحث

Linear Search — البحث الخطي

يفحص كل عنصر بالتسلسل حتى يجد المطلوب أو ينتهي القائمة.

if (arr[i] == target)
return i; // وجدناه
return -1; // لم يوجد
O(n)

Binary Search — البحث الثنائي

للمصفوفات المرتّبة فقط — يُقسّم فضاء البحث للنصف في كل خطوة.

⚠️
شرط مسبق مهم

المصفوفة يجب أن تكون مرتّبة قبل تطبيق Binary Search — فخ امتحاني شائع!

mid = (low + high) / 2
// إذا target < arr[mid] → ابحث يساراً
// إذا target > arr[mid] → ابحث يميناً
O(log n)
الجزء الثاني

لوحة هياكل البيانات

ابحث عن أي هيكل بيانات أو خوارزمية بالاسم

Hash Table
جدول التجزئة

التعريف: هيكل يربط مفاتيح (Keys) بقيم (Values) عبر دالة تجزئة (Hash Function) تحسب فهرساً في مصفوفة.

دالة التجزئة — Division Method
index = key % size
تجزئة النصوص
index = (sum of ASCII values) % size

معالجة التصادم:

  • Chaining: كل خلية تحوي Linked List — عناصر متعددة في نفس الفهرس تُضاف للقائمة
  • Linear Probing: إذا الخلية محجوزة، ابحث عن التالية: (index + 1) % size
📋
مثال امتحاني — Chaining

أدرج [15, 25, 35] في جدول حجم 10:
15 % 10 = 5 | 25 % 10 = 5 (→ في نفس القائمة) | 35 % 10 = 5

Queue
الطابور (Queue)

التعريف: هيكل خطي — الإدراج من الخلف (rear) والحذف من الأمام (front).

🔑
المبدأ الأساسي

FIFO — First In, First Out: أول من يدخل أول من يخرج

العمليات الأساسية:

العمليةالوظيفة
push() / enqueue()إضافة للخلف
pop() / dequeue()حذف من الأمام
front()عرض العنصر الأول
size()عدد العناصر
التنفيذ بالمصفوفة
front = rear = -1 (تهيئة)
ممتلئ إذا: rear == SIZE - 1
BST
شجرة البحث الثنائية

التعريف: هيكل شجري — اليسار يحوي مفاتيح أصغر، اليمين يحوي مفاتيح أكبر.

بنية العقدة: بيانات + مؤشر يسار + مؤشر يمين

⭐
نقطة امتحانية مضمونة

Inorder Traversal (يسار ← جذر ← يمين) دائماً يعطي العناصر بترتيب تصاعدي.

أنواع المرور (Traversal):

  • Inorder: Left → Root → Right
  • Preorder: Root → Left → Right
  • Postorder: Left → Right → Root
Heap
الكومة (Heap)

Max Heap: الجذر = الأكبر — كل أب أكبر من أبنائه.

Min Heap: الجذر = الأصغر — كل أب أصغر من أبنائه.

تمثيل الـ Heap بالمصفوفة
Left Child = 2 * index + 1
Right Child = 2 * index + 2
Parent = (index - 1) / 2
💡
الحدس

إضافة عنصر → ضعه في النهاية وـ"ارفعه" (Swap Up). حذف الجذر → استبدله بالأخير وـ"أنزله" (Swap Down).

Graph
الرسم البياني (Graph)

التعريف: مجموعة عقد (Vertices) مرتبطة بحواف (Edges).

التمثيل: قائمة الجوار (Adjacency List) — vector<vector<int>> — كل فهرس يحوي قائمة جيرانه.

🔵 DFS — Depth First Search

يستكشف بعمق — يذهب لأبعد نقطة قبل الرجوع (Backtracking).

المسار: 1 → 2 → 4 → 5 → 3

يستخدم: Stack
🟢 BFS — Breadth First Search

يستكشف بعرض — يزور كل جيران المستوى الحالي قبل الانتقال.

المسار: 1 → 2 → 3 → 4 → 5

يستخدم: Queue
📌
نقطة مضمونة

DFS → Stack | BFS → Queue — هذا الفرق يظهر في كل امتحان.

الحسابات:

  • 15 % 10 = 5 → الخلية 5: [15]
  • 25 % 10 = 5 → الخلية 5: [15, 25] (Chaining)
  • 35 % 10 = 5 → الخلية 5: [15, 25, 35]
  • 5 % 10 = 5 → الخلية 5: [15, 25, 35, 5]
  • 45 % 10 = 5 → الخلية 5: [15, 25, 35, 5, 45]

جميعها تصطدم في الخلية 5 — Chaining يحل التصادم بالـ Linked List.

Chaining: كل خلية تحوي Linked List — عند التصادم يُضاف العنصر لقائمة نفس الخلية. لا حد للعناصر في خلية واحدة.

Linear Probing: عند التصادم، يبحث عن أول خلية فارغة باستخدام: index = (index + 1) % size

الفرق العملي: Chaining لا يملأ الجدول، Linear Probing يملأه ويسبب التجمّع (Clustering).

  • 10 % 7 = 3 → الخلية 3: [10] ✓
  • 20 % 7 = 6 → الخلية 6: [20] ✓
  • 30 % 7 = 2 → الخلية 2: [30] ✓

لا تصادم هنا. مثال على تصادم: لو أضفنا 17 → 17%7=3 (محجوزة) → (3+1)%7=4 → الخلية 4.

معادلة الأب: Parent = (index - 1) / 2

مثال: العقدة في الفهرس 5 → أبوها في الفهرس (5-1)/2 = 2

المعادلات الكاملة:

  • ابن يسار = 2 * index + 1
  • ابن يمين = 2 * index + 2
  • الأب = (index - 1) / 2

نعم، هي Max Heap صالحة. تحقق من كل أب وأبنائه:

  • فهرس 0 (90): ابن يسار [1]=75, ابن يمين [2]=80 → 90>75 و 90>80 ✓
  • فهرس 1 (75): ابن يسار [3]=30, ابن يمين [4]=60 → 75>30 و 75>60 ✓
  • فهرس 2 (80): ابن يسار [5]=50, ابن يمين [6]=20 → 80>50 و 80>20 ✓

كل أب أكبر من أبنائه → Max Heap صالحة ✓

الخطوات:

  1. أضف 95 في النهاية: [90, 75, 80, 30, 60, 50, 20, 95] — الفهرس 7
  2. أبو الفهرس 7 = (7-1)/2 = 3 → arr[3] = 30. 95 > 30 → Swap
  3. [90, 75, 80, 95, 60, 50, 20, 30] — الآن 95 في الفهرس 3
  4. أبو الفهرس 3 = (3-1)/2 = 1 → arr[1] = 75. 95 > 75 → Swap
  5. [90, 95, 80, 75, 60, 50, 20, 30] — الآن 95 في الفهرس 1
  6. أبو الفهرس 1 = 0 → arr[0] = 90. 95 > 90 → Swap
  7. [95, 90, 80, 75, 60, 50, 20, 30] ✓

Inorder Traversal — ترتيب الزيارة: يسار ← جذر ← يمين

على الشجرة [جذر=8, يسار=4, يمين=12, ...]: Inorder = 2, 4, 6, 8, 12, 15

هذه الخاصية مضمونة رياضياً بسبب تعريف BST نفسه — اليسار دائماً أصغر، اليمين دائماً أكبر.

الإدراج:

  • 50 → جذر
  • 30 < 50 → يسار 50
  • 70 > 50 → يمين 50
  • 20 < 50 < 30 → يسار 30
  • 40 < 50، 40 > 30 → يمين 30
  • 60 > 50، 60 < 70 → يسار 70
  • 80 > 50، 80 > 70 → يمين 70

Inorder (L-Root-R): 20, 30, 40, 50, 60, 70, 80 ← مرتّب تصاعدياً ✓

الاستراتيجيةبعمق — يذهب لآخر نقطةبعرض — مستوى بمستوى
هيكل البياناتStackQueue
يجد أقصر مسار؟لانعم (مسار غير موزون)

BFS يستخدم Queue:

  1. ابدأ: Queue=[1], Visited={1}
  2. أخرج 1، أضف جيرانه [2,3]: Queue=[2,3], Visited={1,2,3}
  3. أخرج 2، أضف جيرانه [4,5]: Queue=[3,4,5], Visited={1,2,3,4,5}
  4. أخرج 3، أضف جيرانه [6]: Queue=[4,5,6], Visited={1,2,3,4,5,6}
  5. أخرج 4 (لا جيران جدد)، أخرج 5، أخرج 6

ترتيب الزيارة: 1, 2, 3, 4, 5, 6

DFS يستخدم Stack:

  1. Stack=[1] → زُر 1، أضف [3,2]: Stack=[3,2]
  2. أخرج 2 → زُر 2، أضف [5,4]: Stack=[3,5,4]
  3. أخرج 4 → زُر 4 (لا جيران): Stack=[3,5]
  4. أخرج 5 → زُر 5 (لا جيران): Stack=[3]
  5. أخرج 3 → زُر 3، أضف [6]: Stack=[6]
  6. أخرج 6 → زُر 6: Stack=[]

ترتيب الزيارة: 1, 2, 4, 5, 3, 6

Stable Sort: يحافظ على الترتيب النسبي للعناصر المتساوية.

Unstable Sort: قد يغيّر الترتيب النسبي للعناصر المتساوية.

  • Stable: Bubble Sort, Insertion Sort, Merge Sort, Counting Sort
  • Unstable: Selection Sort, Quick Sort

مثال: [3a, 1, 3b, 2] → Stable يبقي 3a قبل 3b دائماً. Unstable قد يقلبهما.

maxVal = 9، حجم count = 10

1. عدّ التكرارات: count = [0,2,1,2,1,2,1,0,0,1]

(0→0, 1→2, 2→1, 3→2, 4→1, 5→2, 6→1, 9→1)

2. تراكمي: count = [0,2,3,5,6,8,9,9,9,10]

3. النتيجة: [1, 1, 2, 3, 3, 4, 5, 5, 6, 9] ✓

Adjacency List (vector<vector<int>>):

adj[1] = [2, 3]
adj[2] = [1, 4]
adj[3] = [1, 4]
adj[4] = [2, 3]

كل عقدة تخزّن قائمة جيرانها. هذا أكثر كفاءةً من Adjacency Matrix للرسوم البيانية المتفرقة (Sparse Graphs).

BFS يعمل على مبدأ FIFO — يعالج العقد بترتيب وصولها. أول عقدة تُكتشف أول عقدة تُعالَج.

هذا يضمن أننا نزور كل عقد المستوى الحالي قبل الانتقال للمستوى التالي — وهو تعريف BFS بالضبط.

لو استخدمنا Stack (LIFO) بدلاً منه، سنحصل على DFS تلقائياً!

  1. low=0, high=6 → mid=3 → arr[3]=7 == target → وُجد في الفهرس 3 ✓

مثال أصعب — البحث عن 11:

  1. low=0, high=6 → mid=3 → arr[3]=7 < 11 → ابحث يميناً → low=4
  2. low=4, high=6 → mid=5 → arr[5]=11 == 11 → وُجد في الفهرس 5 ✓

Preorder (Root-Left-Right): 8, 4, 2, 6, 12

Inorder (Left-Root-Right): 2, 4, 6, 8, 12 ← مرتّب

Postorder (Left-Right-Root): 2, 6, 4, 12, 8

حيلة: Preorder يبدأ بالجذر دائماً. Postorder ينتهي بالجذر دائماً.

الشرط: المصفوفة يجب أن تكون مرتّبة تصاعدياً (أو تنازلياً) قبل تطبيق Binary Search.

لو لم يتحقق: الخوارزمية قد ترجع -1 (لم يوجد) حتى لو العنصر موجود فعلاً — نتيجة خاطئة تماماً.

الحل: رتّب أولاً (Merge Sort مثلاً O(n log n)) ثم ابحث بـ Binary Search O(log n).

  • enqueue() / push(): O(1) — نضيف في rear فقط
  • dequeue() / pop(): O(1) — نحذف من front فقط
  • front(): O(1) — نقرأ arr[front] مباشرة
  • size(): O(1) — نحسب rear - front + 1

جميع عمليات Queue الأساسية تعمل بـ O(1) — هذا ميزة رئيسية لهيكل الطابور.

Radix Sort يرتّب رقماً رقماً من الأقل أهمية (آحاد) للأكثر (مئات) باستخدام Counting Sort في كل مرحلة:

مرحلة الآحاد: [170, 90, 802, 2, 24, 45, 75, 66]

مرحلة العشرات: [2, 802, 24, 45, 66, 170, 75, 90]

مرحلة المئات: [2, 24, 45, 66, 75, 90, 170, 802] ✓

Max Heap: الجذر = الأكبر. كل أب ≥ أبنائه. يُستخدم لاستخراج أكبر عنصر بسرعة (Priority Queue للأولوية العليا).

Min Heap: الجذر = الأصغر. كل أب ≤ أبنائه. يُستخدم لاستخراج أصغر عنصر بسرعة (Dijkstra Algorithm).

في كلا الحالتين: الإدراج والحذف بـ O(log n)، الوصول للأقصى/الأدنى بـ O(1).

الطريقة: جمع قيم ASCII لكل حرف ثم تطبيق modulo.

'a' = 97, 'b' = 98, 'c' = 99
sum = 97 + 98 + 99 = 294
hash("abc") = 294 % size

إذا size=10: hash("abc") = 294 % 10 = 4
